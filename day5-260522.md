| 의미 | 객체를 참조할 참조 변수를 선언하는 것 | 실제로 객체를 생성하는 것 (인스턴스화) |

| 메모리 | \*\*스택 (Stack) 영역\*\*에 생성             | \*\*힙 (Heap) 영역\*\*에 생성 |

| 방법 | 클래스 타입의 참조 변수를 선언함        | `new` 키워드를 이용해 실제 객체를 생성함 |



필드의 생명주기

&nbsp;`new` 연산자로 객체가 Heap 메모리에 생성될 때 함께 생성

객체가 메모리에서 사라질 때(Garbage Collector에 의해) 함께 소멸

\- 즉, 객체의 생명 주기와 필드의 생명 주기는 동일함



구분	필드 (Field)	지역 변수 (Local Variable)

선언 위치	클래스 블록 내부	메서드 또는 생성자 내부

저장 위치	Heap 메모리	Stack 메모리

초기화	자동으로 기본값 할당	반드시 수동 초기화 필요

접근 범위	클래스 전체에서 사용 가능	선언된 블록 {} 내부에서만 가능

**추상클래스**

* 추상 메소드 - 본문이 없는 메소드
* 추상 메소드를 선언하면 해당 클래스는 반드시 추상 클래스여야 함

* 추상 클래스는 실무에서 거의 보이지 않음.. ㅎ
* 추상 클래스는 추상 메소드, 일반 메소드 모두 가질 수 있음



#### **추상클래스와 인터페이스의 차이**



추상 클래스는 추상 메소드, 일반 메소드 모두 가질 수 있음 - 자식 클래스에 공통된 속성 혹은 기능을 상속해주고(extends) 싶음 (언제든지 가져다 쓸 수 있음)



인터페이스는 추상 메소드만 모아 놓은 것 - 구현체에서 인터페이스에서 선언된 추상메소드에 대한 구현(implements)이 반드시 강제됨.  오버라이딩 



추상 클래스는 하나만 상속 가능하지만

인터페이스는 여러개 구현 가능



스프링에서는

인터페이스를 훨씬 많이 써요. 바로 OCP 때문에!

구현체가 바뀌어도 비즈니스 로직은 안 건드려도 됨

UserRepository repo = new MySQLUserRepository();

UserRepository repo = new RedisUserRepository();  // 갈아끼우기만 하면 됨



#### **결합도를 낮추기 위한 방법**

결합도란, 클래스 간의 의존 정도를 의미합니다. 

결합도가 높을수록 한 클래스가 변경될 때 다른 클래스도 함께 수정해야 하는 상황이 발생합니다.

결합도를 낮추는 수단이 바로 업캐스팅 + 인터페이스이고,

그걸 원칙으로 표현한 게 OCP(개방-폐쇄 원칙) 이에요!

구현체만 갈아끼우면 비즈니스 로직 자체는 건드리지 않아도 된다!

**에러 vs 예외**

에러란 코드로 해결할 수 없는 인프라/서버 등 외부 요인으로 인한 문제

예외란 코드 수정으로 해결할 수 있는 프로그램 내부적 문제



**throwable**은 자바에서 에러와 예외를 모두 받는 최상위 클래스

모든 예외는 **Exception** 타입으로 업캐스팅 가능



**예외의 종류**

* (1) 일반 예외

코드 짤때 발생하는 예외



* (2) 실행 예외

실행시 발생함

**RuntimeException**이 수퍼 클래스



**예외를 다루는 2가지 방법**

* 예외 처리 (try-catch)
* 예외 회피 (throw) - 해당 메서드를 호출한 곳으로 예외가 던져진다

ex) main 



*Exception in thread "main" java.lang.ArithmeticException: / by zero*

        *at Main.main(Main.java:17*)

*/ by zero => **message***

*java.lang.ArithmeticException => **RuntimeException***
