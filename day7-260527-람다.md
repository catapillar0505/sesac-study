# \[1] 인터페이스

"이런 메서드를 가져야 한다"는 규격서



실제 동작은 없고, 껍데기(추상 메서드)만 있다.

그래서 인터페이스는 직접 new 할 수 없어요. 실제 동작이 없으니까요.

인터페이스를 사용하려면?

클래스가 인터페이스를 구현(implements) 해야 합니다. 클래스가 규격서를 받아 실제 동작을 채워넣는 거예요.



interface Animal {

&nbsp;   void eat();    // 추상 메서드 — 몸통({})이 없음

&nbsp;   void sleep();  // 추상 메서드

}



## \[2] 익명 클래스

만약 딱 한 번만 쓸 클래스를 위해 `Dog`라는 이름을 따로 만들기 귀찮다면? 

이름 없이 그 자리에서 바로 만드는 것이 익명 클래스입니다.

컴파일러가 내부적으로 $Anonymous1 같은 이름의 클래스를 자동 생성합니다. 

우리 눈에 이름이 안 보일 뿐, 실제로는 클래스가 만들어집니다.



------------- 인터페이스를 사용하는 방식!!------------------------------

< 일반 클래스 방식>

class Dog implements Animal {

&nbsp; public void eat() {

&nbsp;   System.out.println("먹는다");

&nbsp; }

&nbsp; public void sleep() {

&nbsp;   System.out.println("잔다");

&nbsp; }

}

Animal a = new Dog();



<익명 클래스 방식>

Animal a = new Animal() {

&nbsp; public void eat() {

&nbsp;   System.out.println("먹는다");

&nbsp; }

&nbsp; public void sleep() {

&nbsp;   System.out.println("잔다");

&nbsp; }

};



## \[3] 함수형 인터페이스

추상 메서드가 딱 1개인 인터페이스를 특별히 함수형 인터페이스라고 부릅니다. 람다를 쓸 수 있는 조건이에요.

=> 람다식을 인자로 전달

< 인터페이스>                                      (메소드)

runnable - 인자 주고받기 안하고 실행만함 (run)

consumer  - 인자를 받기만 함 (accept) 

supplier - 인자를 주기만 함 (get) 

function - 인자를 받아서 변환함 (apply)

predicate - 인자를 받아 boolean값 반환 (test)



// 위 각 인터페이스는 곧 람다임 () -> {}

# \[4] 람다 = 익명함수

익명 클래스로 함수형 인터페이스를 구현할 때, 메서드가 1개라 불필요한 코드가 너무 많습니다. 

람다는 이걸 확 줄인 문법적 축약이에요.



람다는 왜 메서드 1개짜리만 가능할까?

메서드가 2개면 컴파일러가 어느 메서드를 구현한 건지 알 수 없기 때문입니다.



<필요할 때>

1. 메서드를 매개변수로 전달
2. 컬렉션 요소 필터링, 매핑



## **\[5] 스트림 API(\*\*\*)**

컬렉션, 배열 - 요소들 - 람다식으로 처리할 수 있는 **" 반복자"**



* 원본에 훼손이 없다
* for문 없음 - 메소드 호출만으로 스스로 반복함
* 내부적으로 메모리 최적화 진행 - 실무에서도 자주 쓰임



* 스트림 파이프라인(3단계)

1. 생성
   .stream()
   // (1) 리스트 선언 후 리스트의 stream 메소드 호출
   // (2) 배열 선언 후 Arrays의 stream 메소드 호출
   // (3) Stream 클래스의 of 메소드 호출
2. 중간 연산
   .filter(Predicate) => true인 것은 계속 쓰고, false는 사용 안함 
   .map(Function)
   .sorted()
   .distinct()
   .limit()
3. 최종 연산

&nbsp;     .forEach(Consumer)

&nbsp;         .collect(Collector)

&nbsp;         .count()

&nbsp;         .allMatch()





&nbsp;   // stream.forEach(member -> System.out.println(member)); --- (1) 

&nbsp;   

&nbsp;   stream

&nbsp;   .filter(member -> member.length() <= 4)

&nbsp;   .forEach(member -> System.out.println(member)); --- (2)

>> (2)만 실행하면 가능한데, (1)을 실행한 뒤에는 (2)는 실행 안됨.

왜냐하면 (1)에서 스트림 다쓰고 닫혔기 때문임. (foreach는 최종연산)
