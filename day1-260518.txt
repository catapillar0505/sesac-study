2. GIT
(1) VCS = 버전 관리 시스템 => 소스코드를 기억하고 있어서(Snapshot) 언제든지 과거 코드로 돌아갈 수 있음 (형상 관리)
(2) 협업 = 여러 사람이 깃허브 원격 저장소에 저장된 파일을 가져와서 작업, 자신의 브랜치에서 작업하기 때문에 다른 사람의 작업물을 해치지 않음
(3) 기타
 	리누즈 토르발스 = 리눅스 개발한 사람이 개발함
	master 브랜치는 인종차별적 언어라서 main 브랜치로 기본값 설정
	바탕화면에 깃 "워킹 디렉토리" 만들면 안됨
	bash = 리눅스용 쉘(명령어를 내릴수 있는 인터페이스)
(4) 보안
	요즘 깃 자격증명 = 비밀번호로는 안되고... ssh key나 PAT 토큰 사용

3. 리눅스 명령어

= 파일 생성
echo "aaaa" >  a.txt
cat a.txt

4. Untracked files
= 추적되지 않았다
= 무대에 올라가지 않은 파일들

5. git add <파일이름>
= 무대에 올린다
= 스테이징
= 추적 대상에 올린다

- git add .에서 . 의뜻?
현재라는 뜻
현재 폴더의 추적안된 모든 것

6. git commit
= 저장
= 지금 상태를 스냅샷(원본이 아님) 찍는다
= 현재 상태를 그대로 저장

git commit하면 나오는 것 = 리눅스 메모장 = vim 편집기가 나옴

vim 명령어 
그냥 a,i,o > 입력모드로 변경
: 콜론 모드
저장이 w, 종료가 q
wq

7. git rm --cached
= 무대 밑으로 내린다
= 추적을 해제한다
= unstage

8. git status
= 깃 변경 상태 확인

9. git의 id
= 모든 변경사항에 대해 달라지는 형상을 id로 구분함

10. working tree란?
= working directory

11. .git 폴더
= 로컬 저장소
깃의 모든 기록이 저장됨

12. modified 상태
저장 이력이 있는 파일을 수정했을때
수정 상태로 변한 파일을 git add를 한번 더 한후 commit 해야함.

13. git log
실제 git id를 확인할 수 있음
--oneline 옵션 : 약식으로 보임
--graph 옵션: 앞단에 어떤 브랜치에서 나왔는지 확인 가능
