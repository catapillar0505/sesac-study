## 제네릭

클래스를 정의할 때 타입을 T로 비워두고, 나중에 그 클래스를 사용할 때 타입을 결정하도록

**타입을 비워두는 문법**



타입이 결정되는 시점

= 컴파일 시점

= 객체 생성 시점

= 구체화 시점



< 순서>

\[ 소스코드 ]         Box<Integer> box = new Box<>();

       ↓

\[ 컴파일 시점 ]      T = Integer 확정

                     타입 체크 수행

                     <Integer> 정보 소거

                     캐스팅 코드 삽입

                     .class 바이트코드 생성

       ↓

\[ 바이트코드 ]       Box box = new Box();

                     (Integer) box.get();



\[JVM]    JVM 실행 엔진이 바이트 코드를 OS에 맞는 기계어로 변환해줌, OS가 기계어를 CPU에 전달, CPU가 기계어 직접 실행

       ↓

\[ 런타임 ]           <Integer> 정보 없음, 그냥 Box로 동작



타입 결정후 제네릭 타입 소거 후 T는 전부 Object로 변환됨 (자동 업캐스팅)

그 후 다시 강제 다운캐스팅 코드 삽입



컴파일 후 바이트코드 = 런타임시에는 제네릭 타입 사라지고 컴파일러가 캐스팅 자동 삽입함



< 쓰는 이유>

타입 안정성

형변환 생략



\- \*\*표기법\*\*

    - 기본적으로 `<>`(Diamond Operator)를 사용

    - `<>` 내부에 데이터 성격에 따라 대문자 한 글자를 관례적으로 사용함

    - 예

        - `<T>`: Type - 객체의 타입

        - `<E>`: Element - 배열의 요소

        - `<K>`: Key

        - `<V>`: Value



\- \*\*특징\*\*

    1. \*\*⚠️ 주의. 참조 타입만 가능\*\*

        - `int`, `double` 같은 기본 타입(Primitive) 사용 불가

        - 대신 \*\*Wrapper 클래스\*\*(`Integer`, `Double`)를 사용해야 함

    2. \*\*멀티 타입 파라미터\*\*

        - 두 개 이상의 타입을 동시에 지정할 수 있음

        - 예: `<K, V>` → `Map<String, Integer>`

    3. \*\*와일드카드 (`?`) 지원\*\*

        - 타입을 정확히 모르거나 제한하고 싶을 때 사용

        - `<? extends T>`: T 또는 T의 자식들만 가능 (상한 제한)

        - `<? super T>`: T 또는 T의 조상들만 가능 (하한 제한)
