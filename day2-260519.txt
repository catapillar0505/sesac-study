< 네이밍 컨벤션>

각 단어의 첫글자만 대문자를 쓰는 방식
PascalCase: 첫글자가 대문자인 경우 
camelCase: 첫글자가 소문자인 경우 (중간에 톡 튀어나온 낙타등처럼)

kebab-case: 하이픈으로 연결하는 경우 (java에서는 지원하지 않음) (꼬치구이처럼 생겨서)
snake_case: 언더바로 연결하는 경우 (바닥에 기어가는 뱀처럼 생겨서)

1. 기본 타입

	비객체타입
	실제 값을 스택 메모리에 저장하는 방식

	기본 타입의 종류 
	= 4가지
	= 논리, 정수, 실수, 문자
	=> boolean / byte(127까지), short, int(21억까지), long(922경)/ double float / char
	byte 크기: -        / 1   2       4                     8             /          4            8   /   2
	
		- 논리 타입이 정해진 바이트 크기를 갖지 않는 이유 
		1비트면 충분한데 CPU가 비트 단위로 못 읽으니까, 
		크기를 애매하게 두고 JVM한테 위임한 것!
		----------------------------------------------
		- char 문자형 타입은 한글자

ram = 메모리 = 바이트 단위로 저장됨

타입의 표현 범위를 넘어가는 값을 저장하려고 하면 overflow 발생
= 최솟값으로 돌아가는 현상
= 제대로 저장되지 않음
ex) byte a = 128을 저장하려고 하면 byte가 -128~127범위 이므로 가장 작은 -128로 회귀함

double 대신 float를 쓸 이유가 없음..!

각 자료형 타입이 사용할 수 있는 최댓값은 코드를 통해 확인가능
.MAX_VALUE

2. 참조 타입
스택 메모리 -> 참조변수의 변수명과 주소값이 저장됨
	참조 변수 = 참조값(메모리 주소)를 저장하는 변수
	16진법 사용 (10부터 A)
힙 메모리 -> 실제 객체가 저장됨

스택 메모리          				힙 메모리
┌──────────────┐    ┌─────────────┐
│ num │  10    │    				│   "asd"     │
│ str │ 0x123  │			──→ │             │
└──────────────┘    └─────────────┘
